# langchain/ollama 翻译
我们需要先配置一个全局的提示词，然后再配置一个用户的提示词，把这两个提示词合并在一起。如果有用户输入数据之后，然后就会产生一个整体的一个prompt给到大模型。

# 缓存
在在计算机界，然后整个的这个缓存的逻辑都是本质上是一样的，就近优先原则。

第一次问的问题和第二次的问题，一模一样，第二次使用的就是缓存。

如何判断第一次和第二次问题意图是一致的呢？简单一些，就是使用一个新的embedding的库，把第一次提出的问题和第二次提出的问题都转成向量值，去对比它们的差异。通过余弦的方式去对比，如果差异小于我们设定的值，说明他们两个问的问题的意图是一致的。然后我们就可以直接输出上一次缓存答案。

nomic-embed-text 是一个“文本向量化模型”（embedding model）
